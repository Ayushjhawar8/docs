# Contributing to tscircuit

tscircuit is organized into several specialized repositories that work together to provide a complete circuit design ecosystem. This guide will help you understand each repository's purpose and how to contribute effectively.

## Core Repositories

### [tscircuit/tscircuit](https://github.com/tscircuit/tscircuit)
React for Circuits - The main monorepo containing the core packages:
- `@tscircuit/builder`: Core circuit building primitives
- `@tscircuit/react`: React components for circuit creation
- `@tscircuit/schematic-viewer`: Schematic visualization
- `@tscircuit/pcb-viewer`: PCB visualization
- `@tscircuit/router`: Auto-routing engine for PCB traces
- `@tscircuit/soup`: Low-level circuit representation format

### [tscircuit/core](https://github.com/tscircuit/core)
Create electronics with TypeScript:
- Core electronics primitives
- Base component definitions
- Circuit validation logic
- Fundamental circuit types

### [tscircuit/builder](https://github.com/tscircuit/builder)
Create 3D CAD models with React:
- 3D component modeling
- PCB visualization
- Component placement
- Board layout tools

### [tscircuit/schematic-symbols](https://github.com/tscircuit/schematic-symbols)
Schematic symbols for tscircuit:
- Standard electrical symbols
- Custom symbol creation
- Symbol library management
- Reusable symbol components

### [tscircuit/snippets](https://github.com/tscircuit/snippets)
Code snippets and examples:
- Common circuit patterns
- Component usage examples
- Best practices
- Quick start templates

### [tscircuit/docs](https://github.com/tscircuit/docs)
The documentation site at [docs.tscircuit.com](https://docs.tscircuit.com):
- Getting started guides
- API reference
- Component documentation
- Tutorials and examples

## Development Setup

1. **Clone the Repository**
```bash
git clone https://github.com/tscircuit/<repository>
cd <repository>
```

2. **Install Dependencies**
```bash
# Install bun if you haven't already
curl -fsSL https://bun.sh/install | bash

# Install project dependencies
bun install
```

3. **Start Development Server**
```bash
# For tscircuit/tscircuit
bun run dev

# For tscircuit/web
bun run dev

# For tscircuit/docs
bun run start

# For converters
bun run build && bun run test
```

## Contributing Guidelines

1. **Choose the Right Repository**
   - Bug fixes go to the repository where the bug exists
   - New components go to tscircuit/core or tscircuit/tscircuit
   - Documentation updates go to tscircuit/docs
   - Web features go to tscircuit/web
   - Symbols go to tscircuit/schematic-symbols
   - Example code goes to tscircuit/snippets
   - CLI improvements go to tscircuit/cli
   - Routing algorithms go to tscircuit/routing
   - Layout algorithms go to tscircuit/autolayout

2. **Create an Issue First**
   - Describe the problem or enhancement
   - Get feedback from maintainers
   - Link to related issues/PRs

3. **Development Process**
   ```bash
   # Fork and clone the repository
   git clone https://github.com/<your-username>/<repository>
   
   # Create a feature branch
   git checkout -b feature/your-feature-name
   
   # Make your changes
   # Run tests and linting
   bun run test
   bun run lint
   
   # Commit with clear message
   git commit -m "feat: add new component for motor control"
   
   # Push and create PR
   git push origin feature/your-feature-name
   ```

4. **Code Quality**
   - Write tests for new features
   - Update documentation
   - Follow existing code style
   - Use TypeScript strictly
   - Add comments for complex logic

5. **Pull Request Guidelines**
   - Reference related issues
   - Include screenshots for UI changes
   - Update documentation
   - Add tests
   - Keep changes focused
